eslint/no-dupe-class-members Correctness â€‹

âœ… This rule is turned on by default.

What it does â€‹

Disallow duplicate class members

Why is this bad? â€‹

If there are declarations of the same name in class members, the last declaration overwrites other declarations silently. It can cause unexpected behaviors.

Examples â€‹

Examples of incorrect code for this rule:

javascript
class A {
  foo() {
    console.log("foo");
  }
  foo = 123;
}
let a = new A();
a.foo(); // Uncaught TypeError: a.foo is not a function

Examples of correct code for this rule:

javascript
class A {
  foo() {
    console.log("foo");
  }
}
let a = new A();
a.foo();

How to use â€‹

To enable this rule in the CLI or using the config file, you can use:

bash
oxlint --deny no-dupe-class-members
json
{
  "rules": {
    "no-dupe-class-members": "error"
  }
}

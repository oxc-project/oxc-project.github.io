jest/valid-title Correctness ‚Äã

‚úÖ This rule is turned on by default.
üõ†Ô∏è An auto-fix is available for this rule for some violations.

What it does ‚Äã

Checks that the titles of Jest blocks are valid.

Titles must be:

  • not empty,
  • strings,
  • not prefixed with their block name,
  • have no leading or trailing spaces.

Why is this bad?

Titles that are not valid can be misleading and make it harder to understand the purpose of the test.

Example ‚Äã

Examples of incorrect code for this rule:

javascript
describe("", () => {});
describe("foo", () => {
  it("", () => {});
});
it("", () => {});
test("", () => {});
xdescribe("", () => {});
xit("", () => {});
xtest("", () => {});

Examples of correct code for this rule:

javascript
describe("foo", () => {});
it("bar", () => {});
test("baz", () => {});

Options ‚Äã

interface Options { ignoreSpaces?: boolean; ignoreTypeOfTestName?: boolean; ignoreTypeOfDescribeName?: boolean; disallowedWords?: string[]; mustNotMatch?: Partial<Record<'describe' | 'test' | 'it', string>> | string; mustMatch?: Partial<Record<'describe' | 'test' | 'it', string>> | string; }

How to use ‚Äã

To enable this rule in the CLI or using the config file, you can use:

bash
oxlint --deny jest/valid-title --jest-plugin
json
{
  "plugins": ["jest"],
  "rules": {
    "jest/valid-title": "error"
  }
}
